Project Assignment 4 - Semantic Definitions

  • Author: 0516027 Chain-An Yu
  • Email: fsh0524@gmail.com

This is a simple LALR(1) parser for the $\mathcal{P}$ language.

Changelog

  • Added the feature of Symbol Table.
  • Fixed memory leak problem.
  • Now the parser can do semantic check to the code.

Abilities

  • Scan the input and parse the tokens, and check if there are any syntactic errors or semantic errors.

    • If there is a syntactic error, it will output the line number and the unmatched token and then abort the program. For example,

    • Once the parser encounters a semantic error, it will output an error message, which contains the line number and the category of the error. For example,

      Note that the semantic errors won’t cause the parser to stop its execution.

  • While parsing the input, the parser will also build a symbol table. You may use pseudocomments to enable/disable the output of symbol table. For example,

  • Support C/C++ style Commands and Pseudocomments.

    • Now support the following pseudocomments: (Note that all options is disable in default.)
    • //T+, //T-: enable/disable Token Output.
    • //S+, //S-: enable/disable Source code Output.
    • //D+, //D-: enable/disable Dumping Symbol Table.

      Dependency

  • Unix-like platform is highly recommended.
  • lex or flex
  • gcc or clang to compile the scanner.
  • yacc

How to Build and Execute?

  • make clean
    • This make sure the diretory is clean.
  • make
    • Compile the scanner and parser, it may output some warnning messages, ignore it first.
  • ./parser <your_p_program.p>
    • Replace <your_p_program.p> with the file which you want to parse.
